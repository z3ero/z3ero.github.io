可迭代对象

  1. 概念： 可以直接作用于 for 循环的对象统称可迭代对象
  • 集合数据类型： list，tuple, dict, set, string 等
  • generator: 包含生成器和带yield的generator function
  • 判断是否为可迭代对象, 使用 isinstance()函数判断
  1. 生成可迭代对象

  • 列表生成式

  • 生成器
    克服了列表生成器的缺点——所有数据都存到内存中
    生成器的元素是一遍循环，一遍推导出来的，节省内存
    (1) 修改列表生成式的[] 为 ()

    (2) 将函数return 改为 yield
    推导算法太复杂，使用列表生成式无法实现时， 可以选择使用函数式生成器

    注意:使用for循环遍历generator时，拿不到generator的return数值；如果想要返回值，必须用next()，捕获无值时的StopInteration错误，返回值包含在错误对象的value属性中(e.value)。

迭代器

  1. 概念
    迭代器是可迭代对象的子集，它不但可以用于for循环，还可以被next()调用。
    可迭代对象不一定可以被next()调用

  2. 可迭代对象转迭代器

    • Iterator对象表示的是一个数据流，它可以被next函数调用返回数据，直到返回StopIteration错误。
    • 与list，tuple等可迭代对象相比，迭代器并不确定序列长度（可以表示无限大的数据流），是为惰性求值而设计的。

python 闭包与装饰器

  1. 闭包

  • 概念：在函数体中定义内部函数，并且使用了外部函数的变量，然后把内部函数给返回，那么这个内部函数就是闭包

  • 优点：避免污染全局环境，这样就可以在函数体外使用函数体中定义的变量 (每个函数体是自己的作用域)

  • 缺点：数据长期驻留在内存中，造成内存极大的浪费。（实际开发中，要避免使用闭包）

  1. 装饰器

  • 装饰：内在不变，只是外在变化了

  • 概念：是一个闭包，把一个函数作为参数然后返回一个替代版函数，本质上就是一个返回函数的函数

  • 作用：在不修改原函数的前提下增加函数的功能，最好的方式是使用装饰器

  1. 使用@将装饰器应用于函数上
    只需要在函数定义前加上@装饰器的名称即可

  2. 带参数的装饰器
    带参数的装饰器应该内部包含两层函数，装饰器的参数在@处传值

  3. 多个装饰器的场景
    装饰时：从距离近的装饰器开始装饰
    执行时：从距离远的装饰器开始执行

  4. 装饰器使用场景

  • 参数、结果检查： 如上面程序对不合法年龄的检查

  • 缓存 （如下，缓存中间结果，避免重复运算）

  • 计数：如使用装饰器计量函数运行次数

    nolocal 关键字: 内部函数的赋值操作 index += 1 会让index变成新的局部变量，无法与外层index绑定，nolocal将index变量标记为非局部，即便赋值，仍然为非局部变量

  • 日志
    函数本身只有自己的功能，在装饰器中添加日志功能

python垃圾回收机制

python 同java一样，采用垃圾回收机制，不同于java, python采用的是引用计数机制为主，标记清楚和分代回收为辅的策略

内存管理

小整数池

python 中小整数[-5, 256]，单个字符这类对象是提前建立好的，所有这个范围内的整数使用同一个对象，也不会有引用计数，不会触发垃圾回收

intern机制

值相同的字符串对象（整数等不可变类型也是）仅仅会保存一次，是共用的。

在创建新的字符串对象时，如果内存中已经存在，则直接引用，避免频繁的创建和销毁内存，提升效率。

为了解决循环引用导致的内存泄露问题，引入标记清除和分代回收机制

标记清除

概念

对象名：不存值，只存储一个内存地址，指向堆
对象值：程序在堆内开辟的内存（垃圾回收的对象）

标记清除（mark-sweep）是一种基于追踪回收（tracing GC）技术实现的垃圾回收算法。
分为两个阶段：

  1. 标记阶段，GC会把所有的『活动对象』打上标记，
  2. 是把那些没有标记的对象『非活动对象』进行回收

原理

对象引用可以表示成一个以对象为节点，引用关系为边的图。如下图：

root 可以看作是一个存放于栈区的对象名，从它开始，进行深度或者广度优先遍历，将可达对象打上标记，上图中，只有节点5是不可达的。

具体流程：

  1. 遍历GC root引用（栈区内的对象名）
  2. 使用DFS或者BFS搜索堆，标记搜索到的对象『活动对象』— 一般使用占内存较小的DFS
  3. 前两步完成之后，遍历堆，找到未被标记的对象『非活动对象』，进行回收

优缺点

标记清除算法主要用于回收容器对象，如dict,list,tuple等，因为容器对象才会造成循环引用

  • 优点: 回收循环引用垃圾，算法简单
  • 缺点: 效率低。清除非活动对象之前，必须顺序扫描整个堆内存，即使只剩下小部分活动对象，也要扫描所有。

分代回收

概念

在CPython中，【引用计数机制】 是实时进行的，但是效率较低的【标记清除】何时执行，对那些对象执行？为了改善标记清除的效率问题，引入【分代回收机制】

分代回收的思想：直觉上，一个对象存活时间越长，依赖引用它的对象会越多，越不容易被回收。

原理

  1. 新创建的对象称为0代
  2. 每执行一次【标记清除】，存活下来的对象代数+1
  3. 代数越高的对象，进行【标记清除】的时间间隔越长

python将对象总共分为三代，对应3个链表，它们的垃圾回收频率与对象存活时间成反比，这个回收频率可以通过gc.set_threshold(threshold0 [, threshold1 [,threshold2]])来设置

当gc中新增的对象数量 - 删除的对象数量 > threshold0, python对第0代进行一次【标记清除】。
当第0代被回收次数 > threshold1, 对第一代对象【标记清除】。
当第1代被回收次数 > threshold2, 对第二代对象 —-
以此类推
