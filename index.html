python垃圾回收机制

Posted on 2019-06-30 | In 编程语言

python一些内存分配特性

  1. 小整数池：python 中小整数[-5, 256]，单个字符这类对象是提前建立好的，所有这个范围内的整数使用同一个对象，也不会有引用计数，不会触发垃圾回收

  2. intern机制： 值相同的字符串对象（整数等不可变类型也是）仅仅会保存一次，是共用的。

  • 在创建新的字符串对象时，如果内存中已经存在，则直接引用，避免频繁的创建和销毁内存，提升效率。

python—知识点一

Posted on 2019-05-28 | In 编程语言

解释器

  • CPython：使用广泛的python解释型编译器
  • IPython：基于CPython的交互式编译器
  • PyPy: 专注于执行速度的python解释器，使用JIT(just-in-time)技术，静态编译和动态解释相结合。JIT通用方法：(1) 标记经常被执行的字节码 （2）将其编译成本地的机器码 (3) 缓存该结果 (4)当同样的字节码再次被执行时，预期编译好的机器编码。
    (JIT技术难做的原因——python语言数据在运行时才确定)
  • JPython：将python代码编译成 java 字节码，再由JVM解释执行
  • IronPython: .Net平台上的解释器，编译成.Net的字节码

PE8命名规范

  • 用4空格而不是tab来缩进
  • 每行字符 < 79
  • 操作符和逗号后面用空格，不要在括号内部使用
  • 类命名：CamelCase, 函数：小写字母+下划线
  • 大段代码（如类和函数之间）用空行间隔

and or

python 的 and 和 or 相当于 C++ 中的 && 和 ||

总结:

  • x or y 若 x 为真, 值为 x, 否则为 y
  • x and y 若 x 为真, 值为y, 否则为 x

    最大递归层数

    按照默认设置为 998 层，这个参数可以通过 sys.setrecursionlimit(n) 来设置

== 和 is

== 比较的值，is比较的是地址

运算符优先级

索引 > 属性访问 > 指数 > 按位取反 > 符号运算符> 乘除 > 加减 > 位移 > 按位与，异或，或 > 比较运算符 > is > in > 逻辑运算符(not, and, or)

  1. 索引： 数组索引，例如 arr[0]
  2. 属性： 类对象的属性
  3. 按位取反：~1 = -2
  4. 符号运算符 值得是正负号

有符号数和无符号数

  1. 有符号数转无符号数

  2. 无符号数转有符号数

  3. 补码
    eg: -1 存储为 0-1， -3 存储为 0-3
    所以 -1 = 111…1111; -3 = 111…1101

编码

  1. ascii 只有 1 字节，只能表示 256 个字符, python2 默认编码
  2. unicode 万国码 任意字符2字节
  3. utf-8 unicode升级版, 中文 3 字节, 英文 1 字节, python3默认编码
  4. gbk 国内版本 中文字符2字节, 英文字符1字节
  5. gbk转utf-8 需要通过 unicode

小数据池

小数据池是一种缓存机制，python将某些数据缓存，使用这些数据不用重新创建对象，而是使用已经创建好的对象

  1. 对于整数: python3 将 -5~256 缓存, 超过这个范围，即使两个变量等值，但是地址不同

  2. 对于字符串: python3 将所有不含特殊字符串的字符串缓存

lambda 表达式

匿名函数: 为解决功能简单的需求而设计的一句话函数

args与*kwargs

  1. *args 位置参数, 接受任意多参数并作为元组传入函数
  2. **kwargs 关键字参数, 接收任意多 key-> value 参数，并作为字典传入函数；必须放在 *args后面

python浅拷贝和深拷贝

  1. 浅拷贝: 指针指向已有地址

  2. 深拷贝: 指针指向新开辟空间

可变类型和不可变类型

  1. 可变数据类型: 列表, 字典
  2. 不可变类型: 数字, 字符串, 元组

python垃圾回收机制

  1. 引用计数
  2. 标记清除
  3. 分代回收

IP代理

  1. 原理：用户与目的地之间的中转站，双方都经由该中转站—Ip代理来交换信息。对爬虫而言，用户将抓取请求发给代理服务器，由代理服务器向目标网站请求，再将爬取信息转发给用户。
  2. 作用：隐藏真实IP, 防止自身IP被爬取网站封禁
  3. 分类：
    • 高度匿名代理——对数据包原封不动转发，目的服务器记录IP为代理服务器IP
    • 普通匿名代理——数据包有改动，服务器有几率追查客户真实IP
    • 透明代理——数据包有改动，且代理服务器会告知目的服务器用户的真实IP
    • 间谍代理——组织用于监控内部人员网络信息而搭设的代理
