uuid模块

  1. 128bit的全局唯一标识符，通常由32字节的字符串表示，保证时间和空间的唯一性（GUID）
  2. 原理：mac地址+时间戳+命名空间+随机数+伪随机数来保证产生的ID唯一性
  3. 作用：随机字符串，可以当做口令使用，比如订单号之类的
  4. 算法：uuid1~5() 每个算法的种子和算法不同

base64

  1. 概述: 对数据进行编码和解码，只是一种通过查表的编码方法，不能用于加密，不够3位用\x00字节在末尾补足，编码后=个数表示补了几位。
  2. 编码原理：
    (1) 对二进制字符串如 b’hello world!’, 每3个字节一组进行处理
    (2) 对3*8=24bit 重新划分为4组，每组6bit, 将6bit转为10进制数(064)
    (3) 根据(2)中的4个0    64数，查表[A-Z, a-z, 0-9, ‘+’, ‘/‘]获得相应的4个字符，就是这3个字节编码后的字符串
  3. 作用:
    适用于小段内容的编码，如数字证书签名，cookie, 网页中传输的少量二进制数据
  4. 异常
    由于base64编码结果中可能出现 ‘+’, ‘/‘ 字符，会造成url歧义
    因此对于url编码，使用 base64.urlsafe_b64encode(str)，相应的，使用 urlsafe_b64decode() 解码

hashlib

  1. hashlib 模块提供了常见的摘要算法，如MD5, SHA1，SHA256, SHA526
  2. 原理：摘要算法(哈希算法，散列算法)：通过一个函数，把任意长度的数据转为一个长度固定的字符串（通常16进制的字符串表示）
    (1) md5： 最常见的摘要算法，速度快，生成32位16进制字符串
    (2) sha1: 结果是160bit, 40位16进制字符串
    (3) sha256: 64位16进制字符串
    (4) sha512: 128位16进制字符串
  3. hash碰撞：因为摘要算法是将无限多的数据映射到有限的集合中，所以会冲突
  4. 应用：用户密码加密，防止脱裤

hmac

  1. 对于同一条数据，key不同得到不同的摘要值
  2. 原理：HMAC算法，用一个key对数据进行杂凑后，再进行hash, key不同，加密后的结果也不同

解释器

  • CPython：使用广泛的python解释型编译器
  • IPython：基于CPython的交互式编译器
  • PyPy: 专注于执行速度的python解释器，使用JIT(just-in-time)技术，静态编译和动态解释相结合。JIT通用方法：(1) 标记经常被执行的字节码 （2）将其编译成本地的机器码 (3) 缓存该结果 (4)当同样的字节码再次被执行时，预期编译好的机器编码。
    (JIT技术难做的原因——python语言数据在运行时才确定)
  • JPython：将python代码编译成 java 字节码，再由JVM解释执行
  • IronPython: .Net平台上的解释器，编译成.Net的字节码

PE8命名规范

  • 用4空格而不是tab来缩进
  • 每行字符 < 79
  • 操作符和逗号后面用空格，不要在括号内部使用
  • 类命名：CamelCase, 函数：小写字母+下划线
  • 大段代码（如类和函数之间）用空行间隔

  • x and y 若 x 为真, 值为y, 否则为 x

    • 高度匿名代理——对数据包原封不动转发，目的服务器记录IP为代理服务器IP

    • 透明代理——数据包有改动，且代理服务器会告知目的服务器用户的真实IP
    • 间谍代理——组织用于监控内部人员网络信息而搭设的代理

    (2) 将函数return 改为 yield
    推导算法太复杂，使用列表生成式无法实现时， 可以选择使用函数式生成器

    注意:使用for循环遍历generator时，拿不到generator的return数值；如果想要返回值，必须用next()，捕获无值时的StopInteration错误，返回值包含在错误对象的value属性中(e.value)。

迭代器

  1. 概念
    迭代器是可迭代对象的子集，它不但可以用于for循环，还可以被next()调用。
    可迭代对象不一定可以被next()调用

  2. 可迭代对象转迭代器

    • Iterator对象表示的是一个数据流，它可以被next函数调用返回数据，直到返回StopIteration错误。
    • 与list，tuple等可迭代对象相比，迭代器并不确定序列长度（可以表示无限大的数据流），是为惰性求值而设计的。
